# Σύστημα OPEKEPE

Το OPEKEPE System είναι ένα τοπικό Python MVP για διαχείριση αγροτικών ενισχύσεων, ελεγκτική επισκόπηση, αποζημιώσεις κρίσεων και τεχνοοικονομική ανάλυση. Δείχνει πώς οι δηλώσεις αιτούντων, τα στοιχεία γης, τα καλλιεργητικά σχέδια, τα οικονομικά δεδομένα, το ιστορικό ελέγχου, οι ρόλοι πρόσβασης και τα τεκμήρια κρίσεων μπορούν να συνδεθούν σε μία ιχνηλάσιμη υπηρεσία.

Το περιβάλλον χρήστη εμφανίζεται με την επωνυμία **OPEKEPE**. Το εσωτερικό Python package παραμένει `agropekepe` και η δοκιμαστική βάση SQLite παραμένει `agroledger.sqlite3`, ώστε οι υπάρχουσες εισαγωγές, εντολές και δοκιμές να συνεχίσουν να λειτουργούν.

## Τρέχουσες Δυνατότητες

- Τοπική πύλη browser από Python HTTP server χωρίς εξωτερικές εξαρτήσεις.
- Dashboard με επωνυμία OPEKEPE και αισθητική εμπνευσμένη από Python/Django.
- Σταθερός επιλογέας γλώσσας `EL / EN` κάτω αριστερά.
- Επιλογή ρόλου κατά τη σύνδεση για `Applicant`, `Admin` και `Auditor`.
- Εγγραφή αιτούντα και έλεγχος δημόσιας ακεραιότητας.
- Ροή μεταφόρτωσης εγγράφων για ταυτότητα, γη, οικονομικά, τραπεζικά στοιχεία και τεκμήρια κρίσης.
- Προβολή δήλωσης γης με ενέργειες Google Maps center, Google Earth KML και GeoJSON.
- Πρόβλεψη καλλιέργειας και τεχνοοικονομική ανάλυση για απόδοση, ενίσχυση, κόστος, καθαρό περιθώριο, όριο αγοράς, προϊόντα και υποπροϊόντα.
- Οικονομική προβολή αιτούντα με αναμενόμενη ενίσχυση, συμψηφισμούς οφειλών, καταβλητέο ποσό και γεγονότα ροής αγοράς.
- Προβολή κρίσης αιτούντα για καιρικά ή άλλα τεκμήρια ζημιάς, εκτιμήσεις κάλυψης και σενάρια αποζημίωσης.
- Σελίδα admin για αιτούντες με σύνολα, παράθυρα εξυπηρέτησης, ουρές φόρτου και λειτουργικές συστάσεις.
- Πίνακας admin για διαχείριση δικαιωμάτων ρόλων.
- Προβολή auditor εστιασμένη σε έγγραφα αιτούντων και σχετικά αντικείμενα οικονομικής ανάλυσης.
- Σελίδα αναφορών για δημιουργία JSON reports από admin/auditor.
- JSON API endpoints και unit/API tests για τον πυρήνα της υπηρεσίας.

## Ρόλοι

### Applicant

Ο αιτών χρησιμοποιεί το σύστημα ως προσωπικό χώρο εργασίας για τη δική του υπόθεση. Μπορεί να δει επισκόπηση, έγγραφα, δήλωση γης, πρόβλεψη καλλιέργειας, οικονομικά και διαχείριση κρίσης. Μπορεί επίσης να υποβάλει έγγραφα και τεκμήρια κρίσης, να ελέγξει την αναμενόμενη ενίσχυση, να δει συμψηφισμούς οφειλών, να εξετάσει το καταβλητέο ποσό και να κατανοήσει τις προβολές εισοδήματος από προϊόντα και υποπροϊόντα.

### Admin

Ο admin έχει πρόσβαση σε όλες τις ενότητες του dashboard: επισκόπηση, αιτούντες, έγγραφα, δήλωση γης, πρόβλεψη καλλιέργειας, έλεγχο, οικονομικά, διαχείριση κρίσης και αναφορές. Διαθέτει επίσης εργαλεία για σύνολα αιτούντων, πλήθος εγγράφων και ελέγχων, οικονομικά και κρίσιμα περιστατικά, συστάσεις κατανομής φόρτου, κατάσταση παραθύρων εξυπηρέτησης και ορατότητα διαχείρισης ρόλων.

### Auditor

Ο auditor περιορίζεται σκόπιμα σε επιφάνειες ελέγχου: επισκόπηση, έγγραφα και οικονομικά. Η προβολή εγγράφων δείχνει αντικείμενα εγγράφων αιτούντων, ονόματα αρχείων, τρόπους ελέγχου και ρίσκο. Η οικονομική προβολή δείχνει σχετικά αντικείμενα ανάλυσης όπως σχέδιο απόδοσης, υπολογισμό ενίσχυσης, συμψηφισμό οφειλής, κρατήσεις πρώτης πώλησης, αξία υποπροϊόντων και μέγιστο όριο αγοράς.

- `docs/architecture.md` - αγγλικές τεχνικές σημειώσεις και API.

- `docs/el/` - ελληνικές εκδόσεις τεκμηρίωσης.
- `tests` - unit και API tests.

## Τοπική Εκτέλεση

Άνοιξε PowerShell στον φάκελο του έργου:

```powershell
cd C:\Users\user\Documents\OPEKEPE-SYSTEM-main
```

Έλεγχος Python:

```powershell
py --version
```

Εκτέλεση tests:

```powershell
$env:PYTHONPATH = "src"
py -m unittest discover -s tests
```

Αρχικοποίηση βάσης SQLite:

```powershell
$env:PYTHONPATH = "src"
py -m agropekepe.cli --database .\agroledger.sqlite3 init-db
```

Εκκίνηση εφαρμογής:

```powershell
$env:PYTHONPATH = "src"
py -m agropekepe.cli --database .\agroledger.sqlite3 serve --host 127.0.0.1 --port 8080
```

Άνοιγμα:

```text
http://127.0.0.1:8080/
```

Health check:

```powershell
Invoke-RestMethod http://127.0.0.1:8080/health
```

## API Endpoints

- `GET /health` - έλεγχος υγείας υπηρεσίας.
- `GET /dashboard/data` - αρχικοποιημένο snapshot dashboard.
- `POST /applicant-screening` - έλεγχος δημόσιας ακεραιότητας αιτούντα.
- `POST /documents` - υποβολή metadata εγγράφου και ανάλυσης ελέγχου.
- `POST /farmers` - δημιουργία farmer/applicant.
- `POST /parcels` - καταχώρηση αγροτεμαχίου.
- `POST /crop-seasons` - καταχώρηση καλλιεργητικής περιόδου.
- `POST /remote-sensing` - καταχώρηση παρατήρησης remote sensing.
- `POST /first-sales` - καταχώρηση πρώτης πώλησης και φόρου.
- `POST /debts` - καταχώρηση οφειλής για συμψηφισμό ενίσχυσης.
- `POST /subsidy-claims/calculate` - ετήσιος υπολογισμός ενίσχυσης.
- `POST /crisis-events` - δήλωση περιστατικού κρίσης.
- `POST /compensation-claims/calculate` - υπολογισμός αποζημίωσης κρίσης.
- `GET /annual-ledger` - ετήσιο ledger παραγωγού.
- `GET /audit/events` - λίστα audit events.

## Όρια MVP

Πρόκειται για τοπικό demo MVP. Δεν περιλαμβάνει ακόμη production authentication, μόνιμους λογαριασμούς χρηστών, πραγματικές διασυνδέσεις με κρατικά μητρώα, πραγματικές ροές πληρωμών ή production-grade γεωχωρική υποδομή.

Προτεινόμενα επόμενα βήματα:

- Αντικατάσταση του demo role selector με authenticated λογαριασμούς και server-side authorization.
- Αποθήκευση αναθέσεων ρόλων και σχέσεων αιτούντα-χρήστη.
- Προσθήκη OpenAPI τεκμηρίωσης.
- Προσθήκη end-to-end UI tests για πλοήγηση ανά ρόλο.
- Διασύνδεση με επίσημα μητρώα, AADE/myDATA, κτηματολογικά συστήματα, καιρικά τεκμήρια και συστήματα πληρωμών.
- Μεταφορά γεωχωρικής αποθήκευσης σε PostGIS για παραγωγική χρήση.
- Προσθήκη ανθεκτικής σε αλλοιώσεις αποθήκευσης audit.
