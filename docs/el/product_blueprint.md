# Προϊόν και Ροές

Το OPEKEPE System είναι ένα τοπικό MVP για πλατφόρμα αγροτικών ενισχύσεων, ελέγχου, αποζημιώσεων κρίσεων και τεχνοοικονομικής επισκόπησης με βάση ρόλους. Δείχνει πώς αιτούντες, διαχειριστές και ελεγκτές μπορούν να δουλεύουν πάνω στο ίδιο αρχείο αιτούντα, βλέποντας διαφορετικές ροές ανάλογα με τα δικαιώματά τους.

English version: [docs/product_blueprint.md](../product_blueprint.md)

## Στόχοι Προϊόντος

- Παροχή καθαρού χώρου εργασίας με επωνυμία OPEKEPE για ροές αγροτικής υποστήριξης.
- Σύνδεση ταυτότητας αιτούντα, δήλωσης γης, εγγράφων, καλλιεργητικών σχεδίων, οικονομικών τεκμηρίων, υπολογισμών ενίσχυσης, τεκμηρίων κρίσης και ιστορικού ελέγχου.
- Παροχή πρακτικών εκτιμήσεων στους αιτούντες για αναμενόμενη ενίσχυση, συμψηφισμούς οφειλών, όριο αγοράς καλλιέργειας, αξία υποπροϊόντων και κρατική κάλυψη μετά από κρίσεις.
- Παροχή λειτουργικής εικόνας στους admins για αιτούντες, παράθυρα εξυπηρέτησης, ουρές εργασίας και δικαιώματα ρόλων.
- Παροχή εστιασμένης προβολής ελέγχου στους auditors για έγγραφα αιτούντων και σχετικά αντικείμενα οικονομικής ανάλυσης.

## Ρόλοι Χρηστών

### Applicant

Ροές αιτούντα:

- Εγγραφή ή σύνδεση ως αιτών.
- Επισκόπηση προσωπικής αίτησης.
- Μεταφόρτωση απαιτούμενων εγγράφων και τεκμηρίων κρίσης.
- Δήλωση γης με ροές βασισμένες σε χάρτη, KML και GeoJSON.
- Επισκόπηση πρόβλεψης καλλιέργειας και τεχνοοικονομικής ανάλυσης.
- Επισκόπηση αναμενόμενης ενίσχυσης, συμψηφισμών, καταβλητέου ποσού, εισοδήματος προϊόντων, εισοδήματος υποπροϊόντων και ορίου αγοράς.
- Υποβολή τεκμηρίων για καιρικά ή άλλα περιστατικά ζημιάς.
- Επισκόπηση εκτιμήσεων κρατικής κάλυψης και αποζημίωσης.

Σελίδες αιτούντα:

- Overview
- Documents
- Land
- Forecast
- Finance
- Crisis

### Admin

Ροές admin:

- Προβολή όλων των αιτούντων στο τρέχον snapshot του συστήματος.
- Έλεγχος συνόλων αιτούντων, πλήθους εγγράφων, audit events, οικονομικών εγγραφών και περιστατικών κρίσης.
- Παρακολούθηση κατάστασης παραθύρων εξυπηρέτησης και φόρτου ουρών.
- Επισκόπηση συστάσεων κατανομής φόρτου για intake, έγγραφα, έλεγχο, οικονομική ανάλυση, περιστατικά κρίσης και αναφορές.
- Πρόσβαση σε κάθε σελίδα dashboard.
- Έλεγχος διαμόρφωσης role-management.
- Παραγωγή JSON reports.

Σελίδες admin:

- Overview
- Applicants
- Documents
- Land
- Forecast
- Audit
- Finance
- Crisis
- Reports

### Auditor

Ροές auditor:

- Έλεγχος αντικειμένων εγγράφων αιτούντων.
- Επιθεώρηση audit mode, ονόματος αρχείου και ρίσκου εγγράφου.
- Έλεγχος σχετικών αντικειμένων οικονομικής ανάλυσης.
- Σύγκριση σχεδίου απόδοσης, ενίσχυσης, συμψηφισμών οφειλής, κρατήσεων πρώτης πώλησης, αξίας υποπροϊόντων και ορίου αγοράς.

Σελίδες auditor:

- Overview
- Documents
- Finance

## Εμπειρία Dashboard

Το dashboard περιλαμβάνει:

- Ορατή επωνυμία OPEKEPE.
- Οπτικό ύφος εμπνευσμένο από Python/Django.
- Σταθερό επιλογέα γλώσσας `EL / EN` κάτω αριστερά.
- Επιλογέα ρόλου κατά τη σύνδεση.
- Πλοήγηση ελεγχόμενη από δικαιώματα.
- Κάρτες, πίνακες και γραφήματα ανά ρόλο.
- Canvas charts για οικονομικά, πρόβλεψη καλλιέργειας, κρίση και audit confidence.

## Κύριες Ροές

### Έλεγχος Αιτούντα

Η εγγραφή αιτούντα ελέγχει έκθεση δημόσιας ακεραιότητας και τοπικές αντιστοιχίσεις δηλώσεων. Το αποτέλεσμα οδηγεί τον αιτούντα σε:

- `off_the_hook`
- `enhanced_audit`

Το audit mode των εγγράφων ορίζεται στη συνέχεια ως:

- `standard_audit`
- `close_audit`

### Έγγραφα

Απαιτούμενες κατηγορίες εγγράφων:

- Ταυτότητα ή φορολογικό πιστοποιητικό
- Τίτλος γης, μίσθωση ή κτηματολογικό απόσπασμα
- Οικονομικά αρχεία και τιμολόγια
- Απόδειξη IBAN
- Τεκμήρια περιστατικού κρίσης

Τα υποβληθέντα έγγραφα λαμβάνουν metadata ρίσκου και ελέγχου.

### Πρόβλεψη Καλλιέργειας και Οικονομικά

Το σύστημα εκτιμά:

- Προβλεπόμενη απόδοση
- Μέγιστη benchmark απόδοση
- Ακαθάριστο εισόδημα καλλιέργειας
- Εισόδημα υποπροϊόντων
- Κόστη εισροών και αγρού
- Ενίσχυση
- Συμψηφισμό οφειλής
- Καταβλητέα υποστήριξη
- Καθαρό περιθώριο
- Όριο αγοράς στη μέγιστη απόδοση
- Γεγονότα ροής αγοράς

### Περιστατικό Κρίσης

Οι αιτούντες μπορούν να υποβάλουν τεκμήρια για ζημιές καλλιέργειας που προκαλούνται από καιρό ή άλλες συνθήκες. Το MVP μοντελοποιεί τεκμήρια όπως:

- Έλλειμμα βροχόπτωσης
- Ίχνος πλημμύρας
- Παγετός
- Άνεμος
- Περίμετρος πυρκαγιάς
- Φωτογραφίες ζημιάς καλλιέργειας
- Σημείωμα γεωπόνου
- Επιτόπιος έλεγχος
- Μείωση απόδοσης
- Πρώτες πωλήσεις και τιμολόγια
- Κόστη επισκευής
- Αξία χαμένης παραγωγής

Η σελίδα κρίσης εκτιμά ακαθάριστη πληρωμή κρίσης, εκτεθειμένη αξία περιουσίας, απώλεια καταστροφής, συλλογική απώλεια εσόδων, καθαρή ενίσχυση μετά από συμψηφισμούς και κατάσταση ελέγχου.

### Λειτουργίες Admin

Η σελίδα Applicants του admin παρέχει:

- Πίνακα όλων των αιτούντων
- Σύνολα αιτούντων
- Μετρήσεις λειτουργικών ουρών
- Προβολή παραθύρων εξυπηρέτησης
- Συστάσεις κατανομής φόρτου
- Γρήγορη πλοήγηση σε παράθυρα ελέγχου

### Αναφορές

Οι admin και auditor χρήστες μπορούν να παράγουν τοπικό JSON report που περιέχει:

- Ταυτότητα αιτούντα
- Κατάσταση ακεραιότητας
- Κατάσταση εγγράφων
- Ευρήματα ελέγχου
- Οικονομική έκθεση
- Κατάσταση κρίσης

## Περιορισμοί MVP

- Το σύστημα είναι τοπικό demo.
- Τα δικαιώματα ρόλων είναι προς το παρόν client-side.
- Οι λογαριασμοί χρηστών δεν αποθηκεύονται μόνιμα.
- Οι αναφορές παράγονται client-side.
- Οι εξωτερικές διασυνδέσεις είναι προσομοιωμένες ή εκπροσωπούνται ως τοπικές έννοιες.

## Επόμενα Βήματα Προϊόντος

- Προσθήκη πραγματικού authentication και μόνιμων χρηστών.
- Μεταφορά δικαιωμάτων ρόλων στο server.
- Αποθήκευση αναθέσεων ρόλων και ιδιοκτησίας αιτούντα.
- Προσθήκη admin workflows ανάθεσης σε auditors.
- Προσθήκη αποθήκευσης αναφορών και κατάστασης έγκρισης.
- Προσθήκη OpenAPI τεκμηρίωσης.
- Προσθήκη end-to-end role tests.
- Διασύνδεση με επίσημα κρατικά μητρώα και συστήματα πληρωμών.
